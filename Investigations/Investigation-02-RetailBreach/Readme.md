# Investigation 02 - RetailBreach Lab

## Overview

This investigation focused on analyzing network traffic from a compromised retail web application using Wireshark. The objective was to identify attacker tactics, analyze suspicious activity, investigate web application exploitation attempts, and understand how unauthorized access was achieved.

## Skills Practiced

* Network Traffic Analysis
* Wireshark Investigation
* Web Application Attack Analysis
* IOC Identification
* Network Forensics

## Tools Used

* Wireshark
* CyberDefenders Lab Environment

## Investigation Process

1. Examined network traffic within the provided PCAP.
2. Investigated suspicious HTTP requests and responses.
3. Identified indicators of malicious activity targeting the web application.
4. Analyzed attacker behavior and interaction patterns.
5. Correlated findings with the attack timeline and observed techniques.

## Key Findings

* Identified suspicious web application activity within network traffic.
* Observed evidence of unauthorized access attempts.
* Extracted indicators associated with attacker actions.
* Analyzed attacker interaction with the target application.
* Reconstructed portions of the attack sequence using network evidence.


## Lessons Learned

* Web application attacks often leave clear traces within network traffic.
* HTTP analysis can reveal attacker behavior and exploitation attempts.
* Indicators of compromise are valuable for future detection and monitoring.
* Network forensics helps reconstruct attack timelines and identify malicious activity.

## Reflection

This investigation strengthened my understanding of web application attack analysis and network forensics. It provided hands-on experience in examining attacker behavior through packet analysis and reinforced the importance of identifying indicators of compromise during incident investigations.
